# DevOps Engineer — Planning Reference

## Principles
1. **Multi-stage builds** — для Docker образов
2. **Health checks** — на всех сервисах
3. **Secrets в secret manager** — никогда в коде
4. **Non-root containers** — всегда
5. **Resource limits** — CPU/memory на всём
6. **Infrastructure as Code** — никакого ручного конфигурирования
7. **Structured logging** — JSON формат

## Code Patterns
No specific code patterns. Follow stack principles above.
