# Site Reliability Engineer — Planning Reference

## Principles
1. **SLOs определены** — availability, latency targets
2. **Error budget** — отслеживается, управляет релизами
3. **Golden Signals** — Latency, Traffic, Errors, Saturation
4. **Actionable alerts** — каждый алерт имеет runbook
5. **Blameless postmortems** — учимся на инцидентах
6. **Chaos engineering** — проверяем устойчивость
7. **Toil reduction** — автоматизируем рутину

## Code Patterns
No specific code patterns. Follow stack principles above.
