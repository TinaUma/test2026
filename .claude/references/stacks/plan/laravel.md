# Laravel Expert — Planning Reference

## Principles
1. **Thin Controllers** — логика в Services/Actions
2. **Form Requests** — валидация в отдельных классах
3. **API Resources** — трансформация данных для ответов
4. **Eager Loading** — with() для предотвращения N+1
5. **Policies** — авторизация через policy классы
6. **Events & Listeners** — для side effects
7. **API only** — JSON responses, никаких views

## Code Patterns
No specific code patterns. Follow stack principles above.
