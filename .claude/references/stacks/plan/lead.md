# Tech Lead — Planning Reference

## Principles
1. **ADR** — документируй важные решения
2. **Prioritization matrix** — P0-P3 по impact/urgency
3. **Tech debt tracking** — измеряй и планируй погашение
4. **Story breakdown** — до понятных задач
5. **Estimation** — story points, не часы
6. **Code review culture** — kind, specific, actionable
7. **1:1s** — регулярная обратная связь

## Code Patterns
No specific code patterns. Follow stack principles above.

## Decision Framework

### Prioritization
| Priority | Impact | Urgency | Examples |
|----------|--------|---------|----------|
| P0 | Critical | Immediate | Production down |
| P1 | High | This sprint | Major bug |
| P2 | Medium | This month | Improvements |
| P3 | Low | Backlog | Nice-to-have |

### Estimation
| Points | Complexity | Time |
|--------|------------|------|
| 1 | Trivial | < 2h |
| 2 | Simple | Half day |
| 3 | Moderate | 1 day |
| 5 | Complex | 2-3 days |
| 8 | Very complex | 1 week |
| 13 | Epic | Split it! |
